Configuración de seguridad para las directivas de cuenta

Al asignar contraseñas, el nivel de control de la contraseña debe coincidir con el nivel de protección requerido. Asigne contraseñas seguras siempre que sea posible. La Figura 1 muestra las pautas para crear contraseñas seguras.

Utilice las directivas de contraseña en directivas de cuenta para aplicar las solicitudes de contraseña. Las configuraciones de la Figura 2 cumplen con los siguientes requisitos:

  • Aplique el historial de contraseñas: El usuario puede reutilizar una contraseña después de que se hayan guardado 24 contraseñas únicas.

  • Duración máxima de contraseña: El usuario debe cambiar la contraseña después de 90 días.

  • Antigüedad mínima de contraseña: El usuario debe esperar un día antes de cambiar una contraseña nuevamente. Esto evita que los usuarios introduzcan una contraseña diferente 24 veces para utilizar una contraseña anterior nuevamente.

  • Longitud mínima de la contraseña: La contraseña debe tener al menos 8 caracteres.

  • La contraseña debe cumplir con los requisitos de complejidad: La contraseña no debe contener el nombre de la cuenta o partes del nombre completo del usuario que excedan dos caracteres consecutivos. La contraseña debe tener tres de las cuatro siguientes categorías: letras mayúsculas, letras en minúsculas, números, y símbolos.

  • Guarde las contraseñas mediante cifrado reversible: Guardar las contraseñas mediante cifrado reversible es esencialmente lo mismo que almacenar versiones de texto no cifrado de contraseñas. Por este motivo, esta directiva nunca debe activarse a menos que los requisitos de aplicaciones sobrepasen la necesidad de proteger la contraseña.

Utilice la Directiva de bloqueo de cuentas en Directivas de cuentas para evitar los intentos de inicio de sesión por fuerza bruta. Por ejemplo, la configuración que se muestra en la Figura 3 permite que el usuario introduzca el nombre de usuario incorrecto o la contraseña cinco veces. Después de cinco intentos, la cuenta queda bloqueada por 30 minutos. Después de 30 minutos, la cantidad de intentos se restablece a cero y el usuario puede intentar iniciar sesión nuevamente. Esta directiva también protegería contra un ataque de diccionario, donde cada palabra en el diccionario se ingresa para intentar obtener acceso.
