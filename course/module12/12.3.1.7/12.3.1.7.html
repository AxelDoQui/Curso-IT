Administración de usuarios

Una tarea de mantenimiento regular para los administradores es crear y eliminar usuarios de la red, cambiar contraseñas de usuarios, o cambiar permisos de usuario.

Lo siguiente es lo que debe hacer al administrar cuentas de usuario local:

  • Crear una cuenta de usuario local

  • Restablecer la contraseña para una cuenta de usuario local

  • Deshabilitar o habilitar una cuenta de usuario local

  • Eliminar una cuenta de usuario local

  • Cambiar el nombre de una cuenta de usuario local

  • Asignar una secuencia de comandos de inicio a una cuenta de usuario local

  • Asignar una carpeta doméstica a una cuenta de usuario local

Existen dos herramientas que se pueden usar para cumplir estas tareas:

  • Control de cuentas de usuario (UAC): Use esto para agregar, eliminar, o para modificar los atributos de los usuarios individuales. Cuando se inicia sesión como administrador, utilice el UAC para realizar las configuraciones para evitar que el código malicioso obtenga los privilegios administrativos.

  • Usuarios locales y administrador de grupos: Se pueden utilizar para crear y administrar usuarios y grupos que se almacenan localmente en una computadora.

Nota: Usted debe poseer privilegios de administrador para administrar a los usuarios.

Para abrir UAC, utilice la siguiente ruta de acceso:

Panel de control > Cuentas de usuario > Administrar otra cuenta

Para abrir los usuarios locales y el administrador de grupos, utilice la siguiente ruta de acceso:

Panel de control > Herramientas Administrativas > Administración de computadora > Usuarios y grupos locales

Al utilizar usuarios y grupos locales, puede limitar la capacidad de los usuarios y grupos de realizar ciertas acciones asignándoles los permisos y derechos. Un derecho autoriza a un usuario a realizar ciertas acciones en una computadora, como realizar copias de seguridad de archivos y carpetas o apagar una computadora. Un permiso es una regla que se asocia a un objeto (generalmente un archivo, una carpeta, o una impresora), y regula qué usuarios pueden acceder al objeto y de qué manera.

Desde la ventana de usuarios locales y grupos, haga doble clic en Usuarios. Existen dos cuentas integradas:

  • Administrador: Esta cuenta está deshabilitada de manera predeterminada. Cuando está habilitada, esta cuenta tiene total control de la computadora, y puede asignar derechos de usuario y permisos de control de acceso a los usuarios según sea necesario. La cuenta de administrador es miembro del grupo Administradores en la computadora. La cuenta de Administrador nunca se puede eliminar o quitar del grupo Administradores, pero puede ser retitulada o desactivada.

  • Invitado: Esta cuenta está deshabilitada de manera predeterminada. Esta cuenta se utiliza por los usuarios que no cuentan con una cuenta en la computadora. De manera predeterminada, la cuenta no requiere una contraseña. Es miembro del grupo predeterminado de los usuarios temporales, que permite que un usuario inicie sesión en una computadora.

Para agregar un usuario, haga clic en el menú Acción y seleccione Usuario nuevo. Se abre la ventana Usuario nuevo, como se muestra en la Figura 2. En esta página puede asignar un nombre de usuario, un nombre completo, una descripción, y opciones de cuenta.

Al hacer doble clic a un usuario o hacer clic con el botón secundario y seleccionar Propiedades abre la ventana Propiedades del usuario, como se muestra en la Figura 3. Esta ventana permite cambiar las opciones del usuario definidas cuando se creó al usuario. Además, permite bloquear una cuenta. La ventana también permite asignar un usuario a un grupo utilizando la ficha Miembro de, o de controlar a qué carpetas el usuario tiene acceso al usar la ficha Perfil.

Nota: También hay otro tipo importante de cuenta denominada Usuario Avanzado. Esta cuenta posee el máximo de la potencia de un administrador, como instalar los programas o cambiar la configuración de firewall, pero no tiene algunos de los privilegios de un administrador, por motivos de seguridad.
