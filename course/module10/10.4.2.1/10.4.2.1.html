Tareas programadas

Los sistemas de computación requieren un mantenimiento preventivo periódico para garantizar un mejor rendimiento. Las tareas de mantenimiento se deben programar y realizar con frecuencia para evitar problemas o para detectar los problemas rápidamente. Para evitar perder las tareas de mantenimiento debido a errores humanos, los sistemas de computación pueden programarse para realizar tareas automáticamente. Dos tareas que se deben programar y ser realizadas automáticamente son los controles de copias de seguridad y de disco.

Es importante programar copias de seguridad para asegurar que los datos importantes no se pierdan debido a una falla de hardware. Cuanto más frecuentes las copias de seguridad, menor es el riesgo de pérdida de datos.

La capacidad de medios de base magnética de mantener los cargos electromagnéticos para almacenar datos se utiliza con el tiempo. Al controlar periódicamente el disco para detectar sectores defectuosos, un administrador puede llegar a ser consciente de la posibilidad de un fracaso, lo que permite la planificación y migración de datos.

Las copias de seguridad y controles de disco generalmente son tareas que consumen tiempo. Un beneficio adicional de las tareas de mantenimiento programadas es que permiten que la computadora realice estas tareas cuando ningún usuario utiliza el sistema. La utilidad de la CLI, cron, puede programar estas tareas durante las horas pico.

Cómo programar una tarea: el servicio de cron

En Linux y en OS X, el servicio cron es responsable de las tareas programadas. Como servicio, cron se ejecuta en segundo plano y ejecuta las tareas en las fechas y horas específicas. Cron utiliza una tabla de planificación denominada tabla de cron que puede editarse con el comando crontab.

La tabla cron es un archivo de texto simple que tiene 6 columnas, con el formato que se muestra en la Figura 1. Una tarea se representa generalmente por un comando, un programa o un script. Para programar una tarea, el usuario agrega una fila a la tabla de cron. La fila nueva especifica el minuto, la hora, el día del mes y el día de la semana en que la tarea se debe ejecutar por el servicio cron. Cuando llega a la hora y la fecha determinadas, se ejecuta la tarea.

La Figura 2 incluye una explicación de los campos de la tabla de cron. La columna del centro muestra el tipo de datos aceptable para los campos.

La tabla de cron que se muestra en la Figura 3 tiene dos entradas. La primera entrada le indica al servicio cron ejecutar el script myFirstTask, ubicado en /myDirectory/, en el primer y el decimoquinto día de cada mes y el lunes, siempre a la medianoche (0h0m). La segunda entrada muestra que el servicio cron ejecutará el script mySecondTask, también ubicado en /myDirectory/, cada miércoles a las 2 h 37 m de la mañana.

Para crear o editar la tabla de cron, utilice el comando crontab – e desde una terminal.

Para mostrar la tabla actual de cron, utilice el comando crontab –l.

Para eliminar la tabla actual de cron, utilice el comando crontab –r.
