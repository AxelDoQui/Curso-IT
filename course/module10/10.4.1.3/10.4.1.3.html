Descripción general de la CLI de Linux y OS X

En Linux y en OS X, el usuario puede comunicarse con el sistema operativo mediante la interfaz de línea de comandos (CLI). Para agregar flexibilidad, los comandos (o las herramientas) que admiten parámetros, opciones y switches, son precedidos generalmente por el carácter de guion (“-”). Las opciones y los switches admitidos por un comando también son ingresados por el usuario junto con el comando.

Un programa llamado shell interpreta los comandos del teclado y los transmite al sistema operativo. Cuando un usuario se conecta al sistema, el programa de inicio de sesión revisa el nombre de usuario y contraseña; si las credenciales son correctas, el programa de inicio de sesión inicia el shell. Desde aquí, un usuario autorizado puede comenzar a interactuar con el SO a través de los comandos basados en texto.

Como se muestra en la Figura 1, los usuarios interactúan con el kernel a través de un shell. En otras palabras el shell actúa como capa de la interfaz entre el usuario y el kernel. El kernel es responsable de asignar tiempo y memoria de CPU a los procesos. También administra el sistema de archivos y las comunicaciones en respuesta a las llamadas del sistema.

La mayoría de los sistemas operativos incluye una interfaz gráfica. Aunque una interfaz de línea de comandos aún está presente, el SO arranca a menudo en la GUI de manera predeterminada, escondiendo la interfaz de línea de comandos del usuario. Una forma de acceder a la interfaz de línea de comandos en un sistema operativo basado en GUI es a través de una aplicación del emulador de terminal. Estas aplicaciones proporcionan acceso del usuario a la interfaz de línea de comandos y se denomina generalmente como algunas variaciones de la palabra terminal. En Linux, los emuladores de terminal comunes son Terminator, eterm, xterm, konsole, terminal de gnome-terminal. El OS X incluye un emulador de terminal llamado Terminal pero varios emuladores de terceros están disponibles.

Linux y sistemas de archivos de OS X

Linux y el OS X tienen sistemas de archivos únicos.

Los sistemas de archivos de uso general en Linux son ext3 y ext4 (tercer y cuarto sistema de archivos extendido). Son los sistemas de archivos con journal que mantienen diarios, o registros, de todos los cambios que se harán en el sistema de archivos. Estos diarios minimizan el riesgo de daños en el sistema de archivos en caso de pérdida de alimentación repentina ya que se pueden utilizar para aplicar los cambios después de restablecer la alimentación. Ambos sistemas de archivos pueden admitir tamaños de archivos grandes de hasta 32 TiB (tebibytes) para ext4.

HFS+ (sistema de archivos jerárquico Plus) es el sistema de archivos de inicio que utiliza el OS X. Como ext3 y ext4, HFS+ también admite los volúmenes en diario. Los volúmenes de HFS+ pueden admitir tamaños de archivos grandes de casi 8 EiB (exbibyte) en el OS X 10.4 o posterior.
