Seguridad

A pesar de los esfuerzos continuos para crear un sistema operativo completamente seguro, todavía se siguen encontrando vulnerabilidades. Los usuarios malintencionados sondean sistemas operativos en busca de vulnerabilidades en el código. Cuando se encuentra una vulnerabilidad, puede utilizarse como base para la creación de un virus o de un software malicioso.

Existen numerosas medidas que se pueden tomar para ayudar a evitar que el software malicioso infecte a un sistema de computación. La más común de estas medidas incluye actualizaciones del sistema operativo, actualizaciones de firmware, antivirus y antimalware.

Actualizaciones de sistema operativo

También conocido como parches, las actualizaciones del SO son lanzadas de forma periódica por las empresas del SO para abordar cualquier vulnerabilidad conocida en sus sistemas operativos. A medida que las empresas tienen programadas las actualizaciones, la versión de las actualizaciones no programadas del SO es común cuando una vulnerabilidad importante se encuentra en el código del SO. Los sistemas operativos modernos alertarán al usuario cuando las actualizaciones estén disponibles para la descarga e instalación, pero el usuario puede buscar actualizaciones en cualquier momento. Actualizaciones de firmware

Retenida generalmente en la memoria no volátil, como ROM o flash, el firmware es un tipo de software diseñado para proporcionar la funcionalidad de bajo nivel para un dispositivo. El firmware se encuentra comúnmente en dispositivos incorporados, como relojes digitales, semáforos y dispositivos domésticos. También es común en las computadoras y periféricos de una computadora, cámaras digitales y teléfonos celulares. El firmware es susceptible a las vulnerabilidades y las vulnerabilidades de software; sin embargo, el proceso para actualizar el firmware puede ser más complicado que con los sistemas operativos. Busque actualizaciones de firmware con el fabricante y actualice el sistema si existen nuevas versiones disponibles.

Antivirus y Antimalware

Generalmente el antivirus y antimalware dependen de firmas de código para funcionar. Las firmas o archivos de firma son archivos que contienen un ejemplo de código utilizado por virus y malware. Según estos archivos de firma, el antivirus y antimalware analizan el contenido de un disco de la computadora comparando los contenidos de los archivos almacenados en el disco con ejemplos almacenados en el archivo de firma. Si se encuentra una coincidencia, el virus o el malware alerta al usuario de la presencia posible de malware.

El malware nuevo se crea y se reversiona cada día; por lo tanto, los archivos de firma de antivirus y programas antimalware se deben actualizar con frecuencia.

Administrador de credenciales de seguridad

Los nombres de usuario, contraseñas, certificados digitales, claves de cifrado son solo algunas credenciales de seguridad asociados a un usuario. Debido a la creciente cantidad de credenciales de seguridad necesarias, los sistemas operativos modernos incluyen un servicio para administrarlas. Las aplicaciones y otros servicios pueden pedir y utilizar las credenciales almacenadas por el servicio del administrador de credenciales de seguridad.

El conjunto de claves Gnome es un administrador de credenciales de seguridad para Ubuntu Linux. Para acceder al conjunto de claves Gnome en Ubuntu Linux, como se muestra en la Figura 4, utilice la siguiente ruta de acceso:

Haga clic en Guion > Buscar Clave > haga clic en Contraseñas y claves.

Llaves es un administrador de credenciales de seguridad para OS X. Para acceder a llaves en OS X, como se muestra en la Figura 5, utilice la siguiente ruta de acceso:

Aplicaciones > utilidades > Acceso de la llave
