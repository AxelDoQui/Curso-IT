Comandos administrativos

Con el objetivo de organizar el sistema y reforzar los límites dentro del sistema, Unix utiliza los permisos de archivos. Los permisos de archivos se integran en la estructura del sistema de archivos y proporcionan un mecanismo para definir los permisos a cada archivo. Cada archivo en los sistemas UNIX trae sus permisos de archivo que definen las acciones que el propietario, o el grupo y otros pueden hacer con el archivo. Los derechos posibles de permisos son Leer, Escribir y Ejecutar. Utilizar el comando Is y el parámetro –l para enumerar información adicional sobre el archivo, considere el resultado del comando ls –l a continuación:

rod@machine: $ ls –l my_awesome_file

-rwxrw-r-- 1 rod staff 1108485 Aug 14 7:34 My_Awesome_File

rod@machine: $

El resultado anterior proporciona mucha información sobre My_Awesome_File.

La primera porción del resultado antes visto muestra los permisos asociados con My_Awesome_File. Los permisos de archivos siempre se muestran en el orden usuario, grupo y otro, que lleva a la conclusión de que My_Awesome_File puede ser:

  • El usuario que posee el archivo puede leer, escribir y ejecutar el archivo. Esto se representa por rwx.

  • El grupo que posee el archivo puede leer y escribir al archivo. Esto se representa por rw-.

  • Cualquier otro usuario o grupo del sistema solo pueden leer el archivo. Esto se representa con r--.

La siguiente porción es el número de enlaces directos al archivo, en este caso que es un numeral 1. Esta porción no está dentro del alcance de este curso.

La tercera porción muestra el nombre de usuario del propietario del archivo. La cuarta parte muestra el nombre del grupo que posee el archivo. En este caso, el usuario rod y el grupo staff tienen un cierto nivel de propiedad sobre el archivo.

La quinta porción muestra el tamaño del archivo en bytes. My_Awesome_File tiene 1108485 bytes o cerca de 1,1 MB.

La sexta porción muestra la fecha y la hora de la modificación más reciente.

La séptima porción muestra el nombre de archivo.

El único usuario que puede anular los permisos de archivos en Unix es el usuario raíz. Al tener el poder de anular los permisos de archivos, el usuario raíz puede escribir a cualquier archivo. Dado que todo se trata como un archivo, el usuario raíz tiene control total sobre el sistema operativo UNIX. El acceso raíz se requiere a menudo antes de realizar el mantenimiento y las tareas administrativas.

Nota: Ya que Linux y OS X se basan en Unix, ambos sistemas operativos están en conformidad con los permisos de archivo Unix.
